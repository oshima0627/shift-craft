# ShiftCraft — シフト作成AIシステム

役割・必要人数・時間帯・各種制約を入力すると、条件を満たす**最適なシフト表を自動生成**するブラウザ完結型のWebアプリです。サーバー・ログイン不要で、データは端末のブラウザ内（localStorage）に保存されます。

飲食店のシフト作成のベストプラクティス（労働法・人件費管理・現場運用）を徹底調査し、その結果を機能として組み込んでいます。**調査レポート: [`docs/research.md`](docs/research.md)**

## 特長

### 基本機能
- 🔢 **役割 × 曜日区分（平日/土/日/祝）× 時間帯** ごとに必要人数を設定
- 🚫 **NGペア**：指定した2人を同じ日に出勤させない
- 🧑‍🏫 **新人だけにしない**：各シフトに経験者を最低N名配置
- 🕒 **詳細な時間帯設定**（開始・終了時刻、深夜営業の日またぎ対応）
- 📅 出勤不可日・希望休 / 出勤上限 / 連勤上限
- ✨ ワンクリックで最適シフト生成、✍️ セルクリックで手動微調整（編集後も自動で再検証）
- ⬇️ CSV出力 / 🖨️ 印刷 / 設定のJSONエクスポート・インポート

### 労働法エンジン（調査に基づく・日本の労基法準拠）
- ⚖️ **法定休憩の自動計算**：実働6h超→45分、8h超→60分（34条）。時間帯ごとに表示し人件費にも反映
- 🌙 **深夜割増25%**（22時〜翌5時、37条）を人件費に自動計上
- 🔞 **18歳未満（高校生等）の保護**：22時以降のシフトに割り当てない（61条）、週40h厳守（60条）
- 🗓️ **週の労働時間上限**：40h（法定）/ 44h（常時10人未満の特例措置対象事業場）を選択
- 😴 **週1休の担保**：週の出勤は最大6日（35条）、連勤上限は既定6日
- ⏰ **勤務間インターバル**：終業→翌始業の休息（既定11h、9〜11hが厚労省目安）。
  **クローピング（遅番→翌早番）を防止**。ハード制約/警告のみを切替可能

### その他の条件を「文章」で入力（AI不使用のルールベース解釈）
「田中と佐藤は同じ日に入れない」「高橋は火曜は休み」「伊藤は週3日まで」「鈴木は遅番に入れない」
「田中は金曜は早番固定」「佐藤は4連勤まで」— などの自然文を自動で構造化ルールに変換して
ソルバーに反映します。解釈できない文はメモとして生成結果に表示されます。

### 公平性（定着率対策）
- ⚖️ 出勤回数の公平化 / **土日祝出勤の公平化** / 希望シフトの尊重
  — それぞれ重みスライダーで優先度を調整

> 📝 調査に基づく人件費計算エンジン（人件費率・人時売上高・深夜割増等、`src/solver/cost.ts` と
> `src/components/CostReport.tsx`）は実装済みですが、現在はUIに表示していません。
> 必要になったら生成タブに1行追加するだけで有効化できます。

## AIは必要？（調査の結論）

**シフト表の生成自体にAI（LLM）は不要です。** シフト作成はナーススケジューリング問題として
確立された**数理最適化・制約充足問題**であり、本システムはハード制約厳守＋ソフト制約スコア化の
ヒューリスティクス（貪欲法＋複数回リスタート）をブラウザ内で実行します。商用シフトソフトも
中身は数理最適化かルールベースです。LLMはハード制約（法令等）の充足を保証できないため、
シフト表を直接作らせる設計は不適切です。

LLMが価値を持つのは「**自由文の条件をルールに変換する入力インターフェース**」の部分のみで、
本システムはそこもルールベースのパーサでカバーしています（任意の複雑な文への対応が必要に
なった場合のみ、LLM APIの追加が有効）。詳細は [`docs/research.md`](docs/research.md) §4。

## 技術スタック

| 領域 | 採用 |
|---|---|
| UI | React 18 + TypeScript |
| ビルド | Vite |
| 状態管理 | Zustand（localStorage自動保存・スキーマ移行対応） |
| スタイル | Tailwind CSS |
| 日付処理 | date-fns |
| 最適化 | 自前TSソルバー（貪欲法＋複数回リスタート、AI/LLM・外部API不使用） |
| 法令検証 | 自前エンジン（生成後・手動編集後に同一基準で検証） |
| テスト | Vitest（ソルバー・法令・人件費・パーサ 43テスト） |

外部通信ゼロでオフライン動作します。

## 使い方

```bash
npm install      # 依存インストール
npm run dev      # 開発サーバー（http://localhost:5173）
npm run build    # 本番ビルド（dist/ に静的出力）
npm run preview  # ビルド結果をプレビュー
npm test         # ユニットテスト
```

`npm run build` で生成される `dist/` は静的ファイルなので、GitHub Pages / Vercel / Netlify 等にそのまま配置できます。

## 操作の流れ

1. **期間**：対象月と祝日を設定
2. **役割**：ホール・キッチンなどを登録
3. **時間帯**：早番・遅番などの時刻を設定（法定休憩・深夜・年少者可否が自動表示）
4. **スタッフ**：担当役割・経験レベル・18歳未満・週上限・希望休を入力
5. **必要人数**：役割×時間帯×曜日区分の人数を設定
6. **条件**：労働法ルール・NGペア・経験者最低数・文章での追加条件・優先度を設定
7. **シフト生成**：ワンクリックで生成 → 法令チェックを確認 → 手動調整 → CSV/印刷

## 制約の分類

- **ハード制約**（必ず守る）: 必要人数の充足*・NGペア・経験者最低数・出勤不可日・
  出勤上限・連勤上限・1人1日1シフト・年少者の深夜禁止・週の労働時間上限・週1休・
  勤務間インターバル（厳守設定時）・カスタム条件（曜日NG/時間帯NG・限定/週N日/N連勤/曜日固定）
- **ソフト制約**（できるだけ守る・重み調整可）: 出勤回数の公平化・土日祝の公平化・
  希望シフトの尊重・インターバル（警告設定時）・なるべく同じ日に入れるペア

\* 必要人数はスタッフ不足時のみ満たせないことがあり、その場合は「どこが何名不足か」を明示します。

詳細な要件は [`docs/requirements.md`](docs/requirements.md)、調査レポートは [`docs/research.md`](docs/research.md) を参照してください。
