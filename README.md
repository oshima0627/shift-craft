# ShiftCraft — シフト作成AIシステム

役割・必要人数・時間帯・各種制約を入力すると、条件を満たす**最適なシフト表を自動生成**するブラウザ完結型のWebアプリです。サーバー・ログイン不要で、データは端末のブラウザ内（localStorage）に保存されます。

## 特長

- 🔢 **役割 × 曜日区分（平日/土/日/祝）× 時間帯** ごとに必要人数を設定
- 🚫 **NGペア**：指定した2人を同じ日に出勤させない
- 🧑‍🏫 **新人だけにしない**：各シフトに経験者を最低N名配置
- 🕒 **詳細な時間帯設定**（早番/遅番などの開始・終了時刻）
- 📅 出勤不可日・希望休 / 出勤上限 / 連勤上限
- ⚖️ 公平化・希望シフト尊重を**優先度（重み）**で調整
- ✨ ワンクリックで最適シフト生成、**人数不足・警告を明示**
- ✍️ セルをクリックして**手動微調整**
- ⬇️ CSV出力 / 🖨️ 印刷 / 設定のJSONエクスポート・インポート

## 技術スタック

| 領域 | 採用 |
|---|---|
| UI | React 18 + TypeScript |
| ビルド | Vite |
| 状態管理 | Zustand（localStorage自動保存） |
| スタイル | Tailwind CSS |
| 日付処理 | date-fns |
| 最適化 | 自前TSソルバー（貪欲法＋複数回リスタート＋局所探索） |
| テスト | Vitest |

外部通信ゼロでオフライン動作します。

## 使い方

```bash
npm install      # 依存インストール
npm run dev      # 開発サーバー（http://localhost:5173）
npm run build    # 本番ビルド（dist/ に静的出力）
npm run preview  # ビルド結果をプレビュー
npm test         # ソルバーのユニットテスト
```

`npm run build` で生成される `dist/` は静的ファイルなので、GitHub Pages / Vercel / Netlify 等にそのまま配置できます。

## 操作の流れ

1. **期間**：対象月と祝日を設定
2. **役割**：ホール・キッチンなどを登録
3. **時間帯**：早番・遅番などの時刻を設定
4. **スタッフ**：担当役割・経験レベル・上限・希望休を入力
5. **必要人数**：役割×時間帯×曜日区分の人数を設定
6. **条件**：NGペア・経験者最低数・優先度を設定
7. **シフト生成**：ワンクリックで生成 → 手動調整 → CSV/印刷

## 制約の分類

- **ハード制約**（必ず守る）: 必要人数 / NGペア / 経験者最低数 / 出勤不可日 / 出勤上限・連勤上限 / 1人1日1シフト
- **ソフト制約**（できるだけ守る・重み調整可）: 出勤回数の公平化 / 希望シフトの尊重

完全に満たせない場合も「どこがどれだけ不足か」を明示します。

詳細な要件は [`docs/requirements.md`](docs/requirements.md) を参照してください。
