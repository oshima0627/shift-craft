# シフト作成AIシステム 要件定義書（v1.0）

作成日: 2026-07-14

## 1. 目的
役割・必要人数・時間帯・各種制約を入力すると、条件を満たす**最適なシフト表を自動生成**するWebシステム。
管理者1人がブラウザ上で完結して利用する。

## 2. 前提・スコープ

| 項目 | 決定 |
|---|---|
| AIの実現方式 | **最適化エンジン**（制約充足＋ヒューリスティック最適化）。LLMは使わない |
| 利用形態 | **ブラウザ内で完結**。サーバー・DB・ログインなし。データはブラウザ（localStorage）に保存 |
| 対象規模 | **小規模（〜20人 / 月次）** |
| 希望の収集 | **管理者が一括入力**（単一ユーザー） |

### スコープ外（v1では作らない）
- スタッフ各自のログイン・希望入力
- クラウド保存・複数端末同期
- 自然言語（生成AI）による条件入力
- 給与計算・打刻・勤怠実績連携

## 3. マスタデータ

| マスタ | 主な項目 |
|---|---|
| **スタッフ** | 氏名 / 担当できる役割（複数可） / 経験レベル（新人・一般・ベテラン） / 出勤上限（期間内の最大出勤日数） / 連勤上限 / 出勤不可日・希望休 / 希望シフト（任意） |
| **役割（ロール）** | 名称（例: ホール、キッチン、レジ） / 表示色 |
| **シフト時間帯** | 名称（例: 早番 / 遅番） / 開始時刻 / 終了時刻 |
| **期間設定** | 対象月・開始/終了日 / 祝日指定 |

## 4. 必要人数設定
- **役割 × 曜日区分 × シフト時間帯** ごとに必要人数を設定できる。
- 曜日区分：**平日 / 土 / 日 / 祝**（「土日は何人」を満たす）。
- （拡張）特定日だけ人数を上書きできる。

## 5. 制約（システムの核）

### ハード制約（必ず守る）
| # | 制約 |
|---|---|
| H1 | 各日・各シフト・各役割の**必要人数を満たす** |
| H2 | **NGペア**：指定した2人を同じ日に出勤させない |
| H3 | **新人だけにしない**：各シフトに経験者（新人以外）を最低N名（既定1名）配置 |
| H4 | **出勤不可日・希望休**には割り当てない |
| H5 | **出勤上限・連勤上限**を超えない |
| H6 | 同一人物を同じ日に重複配置しない（1人1日1シフトを既定） |

### ソフト制約（できるだけ守る・優先度つき）
| # | 制約 |
|---|---|
| S1 | **出勤回数の公平化**（人による偏りを最小化） |
| S2 | **希望シフトの尊重** |
| S3 | （拡張）推奨ペア「この2人は一緒だと良い」 |

### 「他の条件」への拡張余地
- 制約はハード/ソフトに分類し、ソフトは**重み（優先度）**で調整可能な設計にする。
- これにより将来的な条件追加（例：特定役割の連続配置回避）を吸収する。

## 6. シフト生成（最適化）
- **入力**：マスタ＋必要人数＋制約。
- **アルゴリズム**：貪欲法（スコアリング）＋複数回ランダムリスタート＋局所探索。小規模なら高速・十分な品質。
- **出力**：
  - 生成されたシフト表
  - **未充足スロット**（人数が足りない箇所）の一覧
  - **制約違反・警告**の一覧
  - スタッフ別の出勤回数（公平性の可視化）
- 完全に満たせない場合も「どこがどれだけ足りないか」を明示する（現場運用で重要）。

## 7. 画面・操作
1. スタッフ管理
2. 役割管理
3. シフト時間帯管理
4. 必要人数設定
5. 制約設定（NGペア・経験者数・上限など）
6. シフト生成＆カレンダー/表表示
7. 手動微調整（ドラッグまたはセル編集）
8. 出力（CSV / 印刷）

## 8. 非機能要件
- **オフライン動作**（ブラウザ完結、外部通信なし）
- データは localStorage に自動保存（再訪時に復元）
- レスポンシブ（PC中心、タブレット可）
- 生成は数百ms〜数秒以内（小規模想定）
- 日本語UI

## 9. データ永続化
- ブラウザ localStorage に全設定・生成結果を保存。
- 設定の **JSONエクスポート/インポート**（バックアップ・端末間の手動移行用）。
