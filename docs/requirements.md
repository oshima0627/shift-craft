# シフト作成AIシステム 要件定義書（v1.1）

作成日: 2026-07-14 / v1.1: 飲食店ベストプラクティス調査（[research.md](research.md)）の反映後に更新

## 1. 目的
役割・必要人数・時間帯・各種制約を入力すると、条件を満たす**最適なシフト表を自動生成**するWebシステム。
管理者1人がブラウザ上で完結して利用する。

## 2. 前提・スコープ

| 項目 | 決定 |
|---|---|
| AIの実現方式 | **最適化エンジン**（制約充足＋ヒューリスティック最適化）。LLMは使わない（調査結論: research.md §4） |
| 利用形態 | **ブラウザ内で完結**。サーバー・DB・ログインなし。データはブラウザ（localStorage）に保存 |
| 対象規模 | **小規模（〜20人 / 月次）**。日本の飲食店を主対象、労基法準拠 |
| 希望の収集 | **管理者が一括入力**（単一ユーザー） |

### スコープ外（v1では作らない）
- スタッフ各自のログイン・希望入力
- クラウド保存・複数端末同期
- 自然言語の**LLMによる**解釈（ルールベースの自然文解釈は実装済み。§5参照）
- 給与計算・打刻・勤怠実績連携（人件費**試算**エンジンは実装済みだがUI非表示）

## 3. マスタデータ

| マスタ | 主な項目 |
|---|---|
| **スタッフ** | 氏名 / 担当役割（複数可） / 経験レベル（新人・一般・ベテラン） / **18歳未満フラグ** / 出勤上限 / 連勤上限 / **週の労働時間・出勤日数上限** / 出勤不可日・希望休 / 希望シフト / （時給: UI非表示） |
| **役割（ロール）** | 名称（例: ホール、キッチン、レジ） / 表示色 |
| **シフト時間帯** | 名称 / 開始・終了時刻（日またぎ可）。**法定休憩・実働・深夜時間・年少者可否を自動表示** |
| **期間設定** | 対象月・開始/終了日 / 祝日指定 |

## 4. 必要人数設定
- **役割 × 曜日区分 × シフト時間帯** ごとに必要人数を設定できる。
- 曜日区分：**平日 / 土 / 日 / 祝**（「土日は何人」を満たす）。
- **特定日の上書き**：「この日は◯人」を曜日区分より優先して指定できる（0人=枠なしも可）。

## 5. 制約（システムの核）

### ハード制約（必ず守る）
| # | 制約 | 根拠 |
|---|---|---|
| H1 | 各日・各シフト・各役割の**必要人数を満たす**（不足時は明示報告） | 運用 |
| H2 | **NGペア**：指定した2人を同じ日に出勤させない | 運用 |
| H3 | **新人だけにしない**：各シフトに経験者を最低N名（既定1名）配置 | 運用（調査§3） |
| H4 | **出勤不可日・希望休**には割り当てない | 運用 |
| H5 | **出勤上限・連勤上限**（既定6連勤）を超えない | 労基法35条 |
| H6 | 同日のシフト重複を禁止。分割勤務OFF=1人1日1シフト。分割勤務ON（既定）=時間帯が重ならなければ同日複数可（早番→遅番など） | 運用 |
| H7 | **18歳未満は22時〜翌5時のシフトに入れない** | 労基法61条 |
| H8 | **週の実働時間上限**：40h／特例44h（選択）。年少者は常に40h | 労基法32・60条 |
| H9 | **週の出勤は最大6日**（毎週1日以上の休日） | 労基法35条 |
| H10 | **勤務間インターバル**（既定11h、厳守設定時）。クローピング防止 | 労働時間等設定改善法（努力義務） |
| H11 | **カスタム条件**：曜日NG / 時間帯NG・限定 / 週N日まで / N連勤まで / 曜日固定 | 運用 |

### ソフト制約（できるだけ守る・重み調整可）
| # | 制約 |
|---|---|
| S1 | **出勤回数の公平化** |
| S2 | **希望シフトの尊重**（希望反映は定着率に直結・調査§3） |
| S3 | **土日祝出勤の公平化**（不公平は離職要因・調査§3） |
| S4 | 人件費の抑制（UI非表示・重み0） |
| S5 | 勤務間インターバル（警告設定時） |
| S6 | **なるべく同じ日に入れるペア** |

### その他の条件の入力方法
1. **自然文入力（AI不使用）**：「田中と佐藤は同じ日に入れない」等をルールベースで構造化。
   解釈できない文はメモとして生成結果に表示（LLM連携の拡張ポイント）。
2. **条件メモ**：自動化できない条件の覚え書き。

## 6. シフト生成（最適化）
- **アルゴリズム**：貪欲法（ハード制約厳守＋ソフト制約スコアリング）
  → **修復パス**（未充足スロットへの同日入れ替え・深さ1チェーン移動、最大4パス）
  → **複数回ランダムリスタート**（既定40回、決定的シード）でベスト解を選択。
- **出力**：シフト表 / 未充足一覧 / **法令エラー・警告**（人数不足と区別して表示）/ スタッフ別出勤数。
- **手動編集後も同じ検証エンジン**（validateSchedule）で法令・人数・運用ルールを再チェック。

## 7. 画面・操作
1. 期間 / 2. 役割 / 3. 時間帯 / 4. スタッフ / 5. 必要人数（＋特定日上書き）
6. 条件（労働法ルール・NGペア・経験者数・自然文条件・優先度）
7. シフト生成＆グリッド表示・セル編集 / 8. 出力（CSV / 印刷 / JSON入出力）

## 8. 非機能要件
- **オフライン動作**（ブラウザ完結、外部通信なし）
- localStorage 自動保存（スキーマ移行対応: persist version 2）
- 生成は数百ms〜数秒以内（小規模想定）／決定的（同一入力→同一結果）
- 日本語UI／印刷対応／相対パス配信（GitHub Pages等の静的ホスティング可）
